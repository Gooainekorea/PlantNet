# 이슈사항 
- 메타 데이터 처리오류
    CSV와 JSON 처리과정 이해 부족.
    왜 사람들이 CSV로 다뤘는지 알게되었다.
    해결 : json 방식으로 저장해보며 적합한 형태로 수정, json 형식 로드시 딕형태로 맵핑

- 생성방식 오류
    기숙사에서 리눅스로 작업하다 들고왔더니 fork()관련해서 오류났다.
    해결 : 모델학습 부분 함수로 감싸고 train()
        if __name__ == "__main__"에서 실행
        참고 : [plantnet_ML.py](Gooainekorea/PlantNet/PlantNet_ML/plantnet_ML.py) 347줄 참고

- 학습곡선 과적합
    학습곡선이 이상하다. 메타데이터 처리 오류 관련한건줄 알았는데 메타데이터 처리 후에도 해결이 되지 않는다.
    ![test_loss](Gooainekorea/PlantNet/00_과정정리/img/test_loss 22.png)

## json 저장 방식(orient) 종류 및 특징

| orient 옵션 | 특징 및 예시 | 용도 |
| :---- | :---- | :---- |
| split | {"index": \[...\], "columns": \[...\], "data": \[...\]} 구조 | 메타데이터와 데이터 분리 필요 시 |
| records | 레코드 리스트 형태 \[{"col1": val1, "col2": val2}, ...\] | 딕셔너리 리스트 형태, 읽기 쉬움 |
| index | 인덱스를 키로 하는 딕셔너리 | 인덱스 기준 조회에 적합 |
| columns | 컬럼명을 키로 하는 딕셔너리 | 컬럼 기준 빠른 조회용 |
| values | 단순 리스트 형태 | 데이터만 저장, 구조 없음 |
